# 飞书日程采集 — 用户指南

通过 **CalDAV** 读取飞书日历（只读，不会在飞书里修改日程）。

## 1. 获取 CalDAV 账号

**飞书桌面端**（需较新版本）：

1. 点击左上角 **头像**
2. **设置 → 日历 → CalDAV 同步 → 进入设置**
3. 选择设备类型（如「其他设备」），点击 **生成**
4. 记录页面上的 **用户名**、**密码**、**服务器地址**（一般为 `caldav.feishu.cn`）

也可在浏览器打开 [飞书日历](https://feishu.cn/calendar)，头像 → 设置 → 底部 **CalDAV 同步**。

**注意**：

- 密码是 **CalDAV 专用密码**，不是飞书登录密码
- 密码可重新生成；重新生成后需更新 `config.yaml`
- 飞书 CalDAV 通常 **只读**（单向同步到本地查看）

## 2. 写入 config.yaml

```yaml
feishu:
  enabled: true
  caldav:
    server: https://caldav.feishu.cn
    username: "粘贴用户名"
    password: "粘贴密码"
    calendar_id: ""
```

`config.yaml` 已在 `.gitignore` 中，**勿提交 Git**。

## 3. 验证采集

```bash
python scripts/collect_all.py --date today --sources feishu
```

成功时输出类似：`└─ N 条日程`，并生成 `data/raw/{date}/feishu_schedule.json`。

排查连接问题（可选）：

```bash
python scripts/debug_feishu_caldav.py
```

再归并进 digest：

```bash
python scripts/collect_all.py --date today --sources feishu
# 或已跑过其它渠道时：
python scripts/merge_daily.py --date today
```

## 4. 常见问题

| 现象 | 处理 |
|------|------|
| 认证失败 / 401 | 在飞书重新生成 CalDAV 密码，更新 config |
| 403 Forbidden | 确认专用密码未过期；可填 `calendar_id`（见下） |
| 0 条日程 | 当天可能无会议；用有会议的日期测试 |
| 与企微/manual 重复 | 同一会议若在多平台都有，digest 可能各一条；可在 manual 中精简 |

## 5. calendar_id（可选）

标准 CalDAV 发现失败时，可填日历 URL 或路径：

```yaml
feishu:
  caldav:
    calendar_id: "https://caldav.feishu.cn/dav/calendar/xxx/"
```

获取方式：用与配置相同的账号密码探测（将 USER/PASS 换成你的）：

```bash
curl -X PROPFIND "https://caldav.feishu.cn/" -u "USER:PASS" -H "Depth: 0" -H "Content-Type: application/xml" --data "<?xml version=\"1.0\"?><D:propfind xmlns:D=\"DAV:\" xmlns:C=\"urn:ietf:params:xml:ns:caldav\"><D:prop><C:calendar-home-set/></D:prop></D:propfind>"
```

在返回的 `calendar-home-set` / 子日历 href 中找到可用路径。

## 6. 技术说明

- 实现：`collectors/feishu_schedule.py` → `collectors/calendar/caldav_client.py`（`provider=feishu`）
- 协议：CalDAV + iCalendar（VEVENT）
- 与企微共用 CalDAV 客户端，发现逻辑走标准 `calendar-home-set`

更多架构说明见 [技术文档.md](./技术文档.md)。
