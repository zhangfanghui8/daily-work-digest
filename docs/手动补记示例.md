# 手动补记示例

用于记录**没有 Git 提交痕迹**的工作：会议、沟通、评审、临时支持等。

## 文件位置

```
data/manual/YYYY-MM-DD.md
```

例如：`data/manual/2026-06-18.md`

## 写法

每行一条，可选时间前缀：

```markdown
- 10:30 与产品对齐下周需求范围
- 14:00 Code Review：支付模块 PR
- 跟进线上告警，协助运维排查连接池问题
- 16:00 项目周会
```

**规则**：

- 以 `-` 开头的列表项会被解析为一条 event
- 行首 `HH:MM` 可选；不写则默认 `12:00:00`
- 含「会议」「对齐」「评审」「Review」等词会自动打上「会议 / 协作」标签

## 保存后

重新归并当天 digest：

```bash
python scripts/merge_daily.py --date 2026-06-18
```

或一键采集+归并：

```bash
python scripts/collect_all.py --date 2026-06-18
```

## 与企微日程的关系

若已配置企微 CalDAV，会议可能已自动采集。手动补记用于：

- 补充会议里的讨论要点（日程只有标题）
- 记录未建日程的临时沟通
- 删除 digest 里重复项后，在 manual 里写更准确的描述
