# 企微日程采集 — 用户指南

通过 **CalDAV** 读取企业微信日程（只读，不会同步到苹果日历等第三方）。

## 1. 获取账号信息

手机企业微信：

**工作台 → 日程 → 右上角「…」→ 日程设置 → 同步至其他日历**

页面上可看到：

- **账号/用户名**（如 `yourname@corp.wecom.work`）
- **服务器**（页面可能显示 `wecom.work`，见下方说明）
- **密码**：点击 **「获取密码」** 生成专用密码（**不是**企微登录密码）

## 2. 写入 config.yaml

```yaml
wecom:
  enabled: true
  caldav:
    server: https://caldav.wecom.work
    username: "粘贴用户名"
    password: "粘贴密码"
    # 若报 403，可填日历数字 ID（见下文）
    calendar_id: ""
```

**注意**：

- `config.yaml` 已在 `.gitignore` 中，**勿提交 Git**、勿发到群里
- 用户名/密码须来自「同步至其他日历」，不是登录密码
- 手机页写的 `wecom.work` 是给 **Exchange** 用的；本工具用 **CalDAV**，服务器填 **`https://caldav.wecom.work`**

## 3. 验证采集

```bash
python scripts/collect_all.py --date today --sources wecom
```

成功时输出类似：`└─ N 条日程`，并生成 `data/raw/{date}/wecom_schedule.json`。

排查连接问题（可选）：

```bash
python scripts/debug_wecom_caldav.py
```

## 4. 常见问题

| 现象 | 处理 |
|------|------|
| 403 Forbidden | 见下方专节 |
| 0 条日程 | 当天可能确实没有会议；可用有会议的日期测试 |
| 与 manual 重复 | 正常；可在 manual 中删掉已自动采集的会议 |

## 5. 403 Forbidden 怎么办

企微 CalDAV **不允许**对根路径做部分探测；脚本已优先访问 `/calendar/`。

若仍 403，按顺序排查：

1. **重新获取同步密码**（最常见）  
   手机「同步至其他日历」→ 生成新密码 → 更新 `config.yaml`

2. **确认用户名完全一致**  
   格式通常为 `你的账号@企业域名.wecom.work`

3. **配置 calendar_id**（数字 ID）  
   企微日历 URL 形如：`https://caldav.wecom.work/calendar/1688855375594291/`  

   ```yaml
   wecom:
     caldav:
       calendar_id: "1688855375594291"
   ```

   获取方式（将 USER/PASS 换成你的）：

   ```bash
   curl -X PROPFIND "https://caldav.wecom.work/calendar/" -u "USER:PASS" -H "Depth: 1" -H "Content-Type: application/xml" --data ""
   ```

   返回内容里找 `/calendar/一串数字/` 即为 `calendar_id`。

## 6. 技术说明

- 实现：`collectors/wecom_schedule.py` → `collectors/calendar/caldav_client.py`
- 协议：CalDAV + iCalendar（VEVENT）
- 钉钉/飞书后续可复用 `caldav_client.py`，仅换 server 与配置

更多架构说明见 [技术文档.md](./技术文档.md)。
