# 禅道采集 — 用户指南

通过禅道 **REST API** 拉取指定日期与你相关的 **Bug / 需求 / 任务** 变更（L1 元数据），写入 digest 供日报/周报使用。

---

## 普通用户怎么用

对 AI 助手说：

```text
写今天的工作日报
```

**你不需要执行任何脚本。** 若禅道未登录，AI 会问你「要不要禅道 Bug/需求/任务」，并按下面三种方式之一帮你连上。

---

## 最简配置

```yaml
zentao:
  enabled: true
  base_url: https://zentao.your-company.com
  account: "你的禅道账号"   # 可选
```

密码**从不**写入 `config.yaml`，只用于换 token，保存在本地 `data/.zentao_token.json`。

---

## 连接禅道的三种方式

### 方式 A：直接给 token（已有 token 时）

对 AI 说：

```text
禅道 token：粘贴这里
```

AI 会写入本地缓存并重采。**不需要密码。**

---

### 方式 B：不知道 token 怎么拿 — 让 AI 教你

对 AI 说：

```text
不知道怎么获取禅道 token
```

AI 会逐步说明，核心步骤如下：

1. 登录禅道 → 进入 **二次开发** → **API** / **API 调试**（不同版本菜单名可能略有不同）。
2. 找到 **获取 Token** 接口：
   - v2：`POST {你的禅道地址}/api.php/v2/users/login`
   - v1：`POST {你的禅道地址}/api.php/v1/tokens`
3. 在调试台请求体填入：

```json
{"account": "你的账号", "password": "你的密码"}
```

4. 发送后，在响应里找到 `"token"` 字段，复制整段值发给 AI。

> token 通常**短期有效**（常见约数十分钟），过期后重复上述步骤即可。

---

### 方式 C：把账号密码告诉 AI，由 AI 代你获取 token

对 AI 说：

```text
禅道账号 xxx，密码 xxx，你帮我连接
```

AI 会在终端代跑登录、把 token 写入 `data/.zentao_token.json`，**你不需要自己运行任何命令**。

- 密码**不会**写入 `config.yaml`
- 仅用于本次换取 token，之后采集用本地 token 缓存

## 采集内容

| 类型 | 典型日报表述 |
|------|--------------|
| Bug | 新建 / 解决 / 关闭 #1234 … |
| 需求 | 新建 / 更新 / 关闭需求 … |
| 任务 | 完成 / 更新任务 … |

产品/项目范围**自动发现**，无需手填 ID。

---

## 常见问题

| 现象 | 处理 |
|------|------|
| 日报写「禅道采集失败（未登录）」 | 对 AI 说「要禅道数据」，用上面 A/B/C 任一方式连接 |
| Token 过期 | 重新获取 token（方式 B）或让 AI 代登录（方式 C） |
| 401 / 403 | 确认账号有 Bug/需求/任务查看权限 |
| 0 条记录 | 当天可能无你的禅道变更 |

---

## 高级（一般用户可忽略）

AI 检测状态：`python scripts/zentao_auth.py --check`
