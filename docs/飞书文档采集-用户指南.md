# 飞书文档采集 — 用户指南

通过 **飞书搜索 API** 拉取指定日期「**我编辑过的**」云文档 / Wiki 的 **L1 元数据**（标题、链接、类型、编辑时间等），**不拉取全文**。

> 与飞书日程（CalDAV）、飞书 IM（开放平台）相互独立，配置块为 `feishu.docs`。

---

## 普通用户怎么用（推荐）

**你对 AI 助手说：**

```text
帮我连接飞书文档
```

或：

```text
@SKILL.md 配置飞书文档并采集今天的文档
```

**你需要做的只有两件事：**

1. **一次性**：把飞书应用的 `app_id` / `app_secret` 交给管理员填进 `config.yaml`（或请 AI 帮你填）
2. **授权时**：AI 运行命令后会打开浏览器，你在飞书页面点 **「同意 / 授权」** 即可

**不需要**复制回调 URL、不需要懂命令行。其余由 AI（Agent）自动完成。

---

## 技术人员 / Agent 一键命令

```bash
python scripts/feishu_oauth.py --login
```

- 自动打开浏览器  
- 本地接收回调并写入 `data/.feishu_oauth.json`  
- 用户只在飞书页点同意  

若遇 **20027（offline_access 权限不足）**：

```bash
python scripts/feishu_oauth.py --login --no-offline
```

（token 约 2 小时有效，长期用需管理员开通 `offline_access`）

采集：

```bash
python scripts/collect_all.py --date today --sources feishu_docs
```

---

## 1. 采集内容（L1）

| 字段 | 说明 |
|------|------|
| 标题 | 文档名称 |
| 链接 | 飞书文档 URL |
| 类型 | DOC / DOCX / SHEET / WIKI 等 |
| 编辑时间 | 当天内的更新时间 |

输出：`data/raw/{date}/feishu_docs.json` → digest 中 `type: document`。

---

## 2. 管理员一次性配置（开放平台）

1. [飞书开放平台](https://open.feishu.cn/app) → 企业自建应用  
2. **权限管理**：`search:docs:read`（可选 `offline_access` 用于长期自动续期）  
3. **安全设置 → 重定向 URL** 添加：

```text
http://127.0.0.1:8765/callback
```

4. 发布应用  

`config.yaml` 示例：

```yaml
feishu:
  docs:
    enabled: true
    app_id: "cli_xxx"
    app_secret: "xxx"
    token_cache: data/.feishu_oauth.json
    redirect_uri: http://127.0.0.1:8765/callback
```

---

## 3. 常见错误

| 错误 | 原因 | 处理 |
|------|------|------|
| **20029** | 重定向 URL 未配置 | 安全设置添加 `http://127.0.0.1:8765/callback` |
| **20027** | 缺少 offline_access | `--login --no-offline`，或请管理员开通权限 |
| 0 篇文档 | 当天未编辑文档 | 换有编辑记录的日期 |
| token 过期 | refresh_token 失效 | 再运行 `--login` |

---

## 4. 高级方式（一般用户可忽略）

| 命令 | 用途 |
|------|------|
| `--login` | 推荐，一键授权 |
| `--exchange "回调URL"` | 手动粘贴地址栏 URL |
| `--paste-token u-xxx` | 从 API 调试台复制短期 token |

详见 `SKILL.md`「飞书文档授权」章节。
